[STEP 3: 질문 프롬프트]

# AI 프롬프트 모음
각 프롬프트는 그대로 복사하여 사용하세요. 모두 "정답 금지, 질문만 생성" 조건을 포함합니다.

## 1. 질문 루프 프롬프트
```
정답 금지, 질문만 생성. 아래 설계 초안의 약점을 드러내는 질문 5가지를 만들어라.
특히 경계·책임·트랜잭션·이벤트·상태를 빠짐없이 건드려라.
---
<내 설계 요약>
```

## 2. 경계 검증
```
정답 금지, 질문만 생성. 아래 컨텍스트/서브도메인 정의를 읽고, 경계가 섞이거나 누락된 부분을 의심할 질문을 만들어라.
각 질문은 "무엇이 왜" 형태로 작성해 토론을 촉발해라.
---
<컨텍스트 설명>
```

## 3. Aggregate 검증
```
정답 금지, 질문만 생성. 애그리게잇 후보 목록을 기반으로 불변조건이 보호되는지 검증할 질문을 5개 제시해라.
각 질문은 루트 책임/트랜잭션/참조 방식 중 하나를 겨냥해야 한다.
---
<애그리게잇 설명>
```

## 4. 이벤트 검증
```
정답 금지, 질문만 생성. 도메인 이벤트 목록을 검토하고, 이벤트 순서·재시도·소비자 책임 관점에서 취약점을 묻는 질문을 작성해라.
---
<이벤트 목록>
```

## 5. 테스트 생성
```
정답 금지, 질문만 생성. 아래 시나리오/테스트 초안에서 빠진 정상/예외/경계/위험 케이스를 찾기 위한 질문을 만들어라.
각 질문은 특정 조건을 상상하게 만들어야 한다.
---
<테스트 표>
```
